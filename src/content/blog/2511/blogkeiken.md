嗨！我是 KAE $( ´ ▽ \` )ﾉ$

這篇文章不聊抹茶甜點，想來聊聊大家現在看到的「這個網站」是怎麼誕生的~
這篇不完全是技術教學，而是我一路走來的「架站小日記」。如果你也正在思考要不要自架站，或是卡在某個步驟，也許我的經驗能讓你少走幾步彎路。

其實一開始，我只是想找個地方記錄抹茶食譜與留學申請心得。但現成平台像方格子、Medium 的版型限制、SEO 不自由、廣告太干擾，都讓我覺得有點綁手綁腳。

我想要一個完全屬於自己、風格簡潔、沒有廣告（或是之後放可以自己獲利的廣告 XD）的小天地。

於是這場「自架網站」的冒險就開始了！

## 階段一：從「視覺」出發的想像與困境

一開始，我對技術一竅不通。我只知道我想要什麼：
![想法到計劃](/images/2511/blog_keiken_03.png)

* **顏色：** 淡淡的綠色（決定下來後固定統一用 #e7f0e2）搭配乾淨的白色。
* **風格：** 簡約、不要複雜的側邊欄。
* **功能：** 首頁要有我的大頭照、文章列表。

我先試著請 Gemini 幫我構思，它用 HTML 寫出了一個「靜態頁面」的原型。

![第一版初稿首頁排版](/images/2511/blog_keiken_01.png)

![定案版本排版](/images/2511/blog_keiken_02.png)

那時候覺得：「哇！做出來了！」但隨即遇到一個大問題：難道我每發一篇文章，都要手動改 HTML 嗎？

只用純 HTML 維護會變得超麻煩：

* 頁首、頁尾不能統一管理（要一個分頁一個分頁去改）。
* 每次作者簡介要更新 → 要改全部檔案。
* AI 每次重生網頁都有可能生錯。
* HTML 想插入圖片、換圖片都很麻煩。
* 如果我之後有 50 篇文章……我要改 50 次？（崩潰）

所以「我需要的不是一頁一頁的網頁，而是一個會自動產生網頁的系統。」

![模板到專案](/images/2511/blog_keiken_04.png)


## 階段二：使用「自動化工廠」 Astro

後來我請 Gemini 把首頁「模板化」，讓頁首、頁尾、版面能共用。 他說要用 Astro + GitHub Pages。

這是一個很大的觀念轉變。以前我以為網站就是「寫好一個網頁檔案放上去」；但現在我知道了，現代的網站更像是一個「自動化工廠」：

* **GitHub（倉庫）：** 放文章原料（Markdown 檔）與圖片。
* **Astro（工廠）：** 讀取原料 → 自動排版成漂亮的網頁（生成首頁、分類頁、文章頁）。
* **GitHub Actions（自動化機器人）：** 我只要上傳一篇文章，它就會建置新網頁、部署更新網站。

聽起來很完美，對吧？但實作起來全是坑......（淚）。

## 階段三：崩潰的踩坑紀錄 (Debugging)

這段過程真的讓我學到很多，分享幾個我卡關最久的地方，希望能幫到也想嘗試的人：

### 1. 被 AI 呼嚨的站內搜尋

我說：「我想做搜尋器。」 AI 說：「沒問題！」 結果做出來的東西完全搜尋不到任何文章。
經過反覆測試，最後改成「先讀取全部文章資料，再用 JavaScript 在前端比對（client-side search）」，才總算成功讓搜尋框運作。

![網站中的搜尋器](/images/2511/blog_keiken_07.png)


### 2. 文字大小沒有設定好

「所有文章」這個頁面是後來才加做的，結果字體大小竟然跟首頁不同步！標題太大、排版歪掉。 為了統一視覺，我甚至叫 AI 生一份「字體大小清單」的 HTML 給我預覽，我再對照真實網頁，一個一個挑出我要的字級，才把樣式統一回來。

### 3. 文章卡片弄到超崩潰

大家現在首頁看到的「最新文章」、「精選文章」方塊卡片，是我跟 AI 大戰三百回合的結果。 一開始標題字太大，手機版只能看到幾個字就被省略號吃掉，像是標題「外面買不到的特濃抹茶蛋塔！」，手機上只顯示「外面買不到的特...」，誰知道是什麼...

我就這樣鬼打牆式地讓 AI 一直調字級、換行規則、刪 CSS、加 CSS，改了超久終於順眼了。

另外還有圖片比例的問題。我希望：

* Google 搜尋結果顯示 1:1 縮圖。
* 點進首頁或分享到 LINE 時，要顯示 16:9 橫式封面圖。

這需要讓程式根據不同情境抓取不同比例的圖片，也是一番折騰才搞定（大家可以分享這篇文章到 LINE 看看，圖片應該要是漂亮的長方形喔！）。

![後台圖片資料庫](/images/2511/blog_keiken_08.png)

![後台文章資料庫](/images/2511/blog_keiken_09.png)
### 4. 恐怖的 "Build Failed" ❌

最崩潰的時刻，莫過於看著 GitHub Actions 的燈號轉圈圈，最後跳出一個紅色的 Failed。 失敗原因常常是：

* **AI 寫錯程式碼：** 亂碼、少括號、多空格。這種防不勝防，建議大家用 AI 時，同一個功能盡量在同一個聊天室進行，讓 AI 熟悉上下文。雖然它還是會亂掉，這時只能把錯誤訊息整段丟回去給它，望著它處理...或是拿去問別的 AI（如 ChatGPT）交叉比對。
* **Front Matter 格式錯誤：** 文章開頭少了 `---`，或者圖片路徑沒寫好，Astro 就會直接兩手一攤：「我看不懂你的原料，我不蓋了。」然後直接停工。
* **樣式設定（Tailwind CSS）遺失：** 有次改完，樣式全不見，原來是少了核心指令。後來我請 AI 把 CSS 設定分離出來獨立管理，才比較穩定。

### 5. 網站排版破裂

有一次修改完後 CSS 樣式完全消失。後來發現是缺少 Tailwind CSS 的核心指令，所以我請 AI 把 CSS 的設定分離出來，讓他更好管理。

### 6. 文章設定成不公開

為了「想測試版面但不想公開文章」，這在其他平台是一鍵完成的事，在自架站上卻超困難。

* 設成 Draft（草稿）：AI 說這樣無法被建置，所以我根本預覽不到。

最後請 AI 修改邏輯，讓特定標籤的文章「會被建置（可以透過連結預覽），但不會出現在首頁列表中」。光是這個邏輯修改就花了我四五個小時，中間還出錯兩次...

## 階段四：看見「綠色勾勾」的感動

最終修正了無數次檔案、提交了無數次 Commit 之後，終於在某一刻，GitHub Actions 的燈號變成了**綠色的 ✅**。

點開網址，看到我寫的文章、我選的圖片、我設計的排版，全部自動且完美地呈現在螢幕上。那一刻的成就感真的難以言喻！

![成功部署](/images/2511/blog_keiken_10.png)

## 結語：我現在要怎麼上架文章

現在，我經營這個部落格變得超級優雅：

* 在電腦 Word / 記事本寫好文章
* 請 AI 幫我轉 Markdown
* 用 Canva 做封面（或調色）
* 打開 GitHub 網頁版 → 上傳 `.md` + 圖片
* 收工！

雖然架設的過程很痛苦都會覺得 AI 在搞我，但擁有一個 100% 由自己掌控、隨時更改的網站，這一切都值得了。

接下來，我會繼續在這個小天地寫作，請大家拭目以待！
